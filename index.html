kafka使用笔记

2019-02-28

kafa官方建议JDK使用1.8版本；

由于kafka依赖zookeeper实现分布式系统的协调，所以需要先安装zookeeper；

zookeeper

1.下载地址：https://zookeeper.apache.org/releases.html#download

下载完解压即可

2.创建zookeeper配置文件

在zookeeper解压后的目录下找到conf文件夹，进入后，复制文件zoo_sample.cfg，并命名为zoo.cfg

zoo.cfg中一共四个配置项，可以使用默认配置，但是数据存储位置需要更改：

示例：dataDir=D:\\Java\\zookeeper\\zookeeper-3.4.12\\tmp
注意：要用双反斜杠，否则系统找不到路径

3.启动zookeeper

可选步骤：

如果为了启动方便，可以将zookeeper添加到系统变量：

添加系统变量：ZOOKEEPER_HOME=D:\Java\zookeeper\zookeeper-3.4.12
path添加：%ZOOKEEPER_HOME%\bin

如果添加了系统变量，直接cmd输入zkServer即可启动zookeeper，否则进入zookeeper根目录下bin文件夹，按住shift+鼠标右键，选择“在此处打开命令窗口”,然后在打开的命令窗口中输入zkServer即可启动zookeeper。

注意：

  • zookeeper启动之后，cmd窗口不要关闭。
  • zookeeper默认使用2181端口，可以在cmd中telnet localhost 2181 测试zookeeper是否启动

kafka

1.下载地址：https://www.apache.org/dyn/closer.cgi?path=/kafka/2.0.0/kafka_2.11-2.0.0.tgz

下载完解压即可

2.修改kafka的配置文件

进入kafka根目录下的config文件夹下，打开server.properties修改配置，可以使用默认，但是log.dirs需要更改：
示例：log.dirs=D:\Java\kafka\kafka_2.11-2.0.0\kafka-logs
注意：要用双反斜杠，否则系统找不到路径

3.启动kafka

在kafka根目录下，按住shift+鼠标右键，选择“在此处打开命令窗口”,然后在打开的命令窗口中输入以下命令即可启动

.\bin\windows\kafka-server-start.bat .\config\server.properties

注意：

  • 路径前方有.
  • 命令指定了使用server.properties来启动kafka
  • kafka默认使用9092端口，可以在cmd中telnet localhost 9092 测试zookeeper是否启动

4.测试：创建主题

进入Kafka根目录，按下Shift+右键，选择“打开命令窗口”选项，打开命令行，输入：

.\bin\windows\kafka-topics.bat --create --zookeeper localhost:2181 --replication-factor 1 --partitions 1 --topic test

注意：

  • 不用换行
  • 创建的主题名称是“test”
  • 该步骤打开的cmd窗口不要关闭

5.测试：查看主题

进入Kafka根目录，按下Shift+右键，选择“打开命令窗口”选项，打开命令行，输入：

.\bin\windows\kafka-topics.bat --list --zookeeper localhost:2181

6.测试：创建生产者

进入Kafka根目录，按下Shift+右键，选择“打开命令窗口”选项，打开命令行，输入：

.\bin\windows\kafka-console-producer.bat --broker-list localhost:9092 --topic test

注意：

  • 该步骤打开的cmd窗口不要关闭

7.测试：创建消费者

进入Kafka根目录，按下Shift+右键，选择“打开命令窗口”选项，打开命令行，输入：

.\bin\windows\kafka-console-consumer.bat --bootstrap-server localhost:9092 --topic test --from-beginning

注意：

  • 不用换行

Git常用命令总结

2019-02-28

初始化配置相关

  • 查看git的版本信息

    git --version

  • 查看当前登录的用户

    git config --global user.name

  • 查看当前登录的用户的邮箱

    git config --global user.email

    注意：如果在配置信息中没有获取到用户配置，说明没有登录。如果没有登录则只能拉取代码，不能修改，要是使用git，需要告诉git是谁在使用。

  • 设置登录用户

    git config --global user.name 'yourUserName'

  • 设置登录用户的邮箱

    git config --global user.email 'yourEmail'

  • 初始化git仓库

    git init

    注意：在gavin文件夹下创建了一个仓库，此时文件夹下会有一个.git的隐藏文件夹

文件操作相关

  • 创建文件夹

    mkdir gavin

  • 切换到文件夹

    cd gavin

  • 创建文件并写入内容

    echo "hello git" > index.html

    注意：单个箭头>表示覆盖写入，>>表示追加

  • 删除本地文件

    rm yourFileName

查询相关

  • 查看目录

    ls -al

  • 查看文件内容

    cat index.html

  • 查询仓库状态

    git status

  • 比较文件差异（请在git add之前使用）

    git diff 文件名

  • 查看仓库历史记录(详细)：

    git log

  • 查看仓库历史记录(单行)

    git log --pretty=online 或 git log --online

  • 查看所有版本的commit ID

    git reflog

添加相关

  • 添加指定文件到暂存区

    git add 文件名 （或git add -A 添加所有文件到暂存区）

    注意：add前需要先初始化仓库git init

  • 暂存区提交到仓库

    git commit -m "注释信息"

撤销相关

  • 撤销工作区的修改

    git checkout -- 文件名

  • 撤销暂存区的修改

    git reset HEAD 文件名

  • 回退到历史版本

    git reset --hard 该版本ID

  • 回退到上个版本

    git reset --hard HEAD^

    注意：上上版本是HEAD^^，也可用HEAD~2表示，以此类推

分支管理

  • 创建分支

    git branch dev

  • 切换分支

    git checkout dev

  • 创建分支并切换分支

    git checkout -b dev

  • 删除分支

    git branch -d dev

  • 在分支上提交新的版本

    git commit -a -m 'dev1'

  • 合并分支

    git merge dev

  • 分支合并后显示log

    git log  --oneline --graph --decorate

  • 在分支开发过程中遇到其他问题需要切换其他分支

    git　stash   //1.保留写好的内容
git checkout master //2.切换到主分支
git stash apply //3.应用保留下来的内容
git stash drop //4.可选：丢掉保留的内容
git stash pop //5.可选：使用保留的内容，并丢掉

  • 最佳分支

    git rebase

远程仓库

  • 先有本地库，后有远程库情况

    git remote add origin 网站上的仓库地址   //关联本地仓库和GitHub库
git push -u origin master    //第一次将本地仓库推送到GitHub上

  • 现有远程库，后有本地库情况

    git clone 网站上的仓库地址    //从远程库克隆到本地，可选https协议或ssh协议

  • 用本地库更新远程库

    git push origin master    //本地库的改动提交到远程库

  • 用远程库更新本地库

    git pull //更新本地库至远程库的最新改动

生成SSH Key

  • 生成SSH Key

    git-keygen -t rsa -C '你的邮箱@xx.com'

    注意：生成的SSH Key在”C:\用户\用户名.ssh”文件夹下
